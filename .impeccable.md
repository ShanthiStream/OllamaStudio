## Design Context

### Users
Power users — ML engineers, software developers, and AI hobbyists running Ollama locally. Technical audience comfortable with terminals and dashboards. They use this tool to manage models, run benchmarks, chat with LLMs, and monitor system resources. Context ranges from late-night tinkering to production prototyping.

### Brand Personality
**Refined, Powerful, Fluid**. The tool should feel like a premium instrument — confident without being loud, smooth without being slow. Think high-end automotive HUD or a professional audio workstation. Every interaction should reinforce that this is a serious tool made by people who care about craft.

### Aesthetic Direction
- **Dark primary** with a proper light mode toggle — both equally polished
- Ambient depth without glassmorphism (no backdrop-blur crutch)
- Data visualization that feels alive — GPU/CPU as generative art
- Motion that serves function first, delight second
- Typography: Already using Onest (warm geometric) — keep it, build around it
- Avoid: gradient text, side-stripe borders, card-in-card nesting, generic AI palette
- Colors: Deep, rich dark theme. Light theme should feel spacious, not clinical

### Design Principles
1. **Motion with purpose** — every animation communicates state or hierarchy. Nothing decorative. The dashboard charts pulse with real data. Page transitions explain spatial relationships.
2. **Data as material** — hardware metrics aren't just numbers on a card. They're flowing, breathing visualizations that make system state instantly readable.
3. **Instant by default** — search filters 50k items without a blink. Theme switches without a flash. The interface never waits.
4. **Depth through physics** — spring-based drag, natural settling, momentum scrolling. Interactions feel physical, not mechanical.
5. **Dual-nature polish** — dark mode is rich and atmospheric. Light mode is airy and precise. Both get equal care.
